    Píldores

    Convenció de noms

    Aquí tens una descripció de les principals convencions de noms en JavaScript:

    1. Camel Case (cas camell)

    • Variables i funcions s'escriuen amb camelCase, on la primera paraula comença en minúscules i cada paraula subseqüent comença en majúscules.
    • Exemple: totalAmount, calculateSum, userName

    Regla: Comença amb una lletra minúscula i utilitza majúscules per separar les paraules.

    2. Pascal Case (també conegut com UpperCamelCase)

    • Classes i constructors s'escriuen amb PascalCase, que és similar al camelCase, però amb la primera lletra de la primera paraula també en majúscules.
    • Exemple: Person, CarModel, DatabaseConnection

    Regla: Comença amb una lletra majúscula i utilitza majúscules per separar les paraules.

    3. Snake Case

    • No és molt comú a JavaScript, però es fa servir en alguns altres llenguatges (com Python o bases de dades) o per noms de fitxers. En aquest cas, les paraules es separen amb guions baixos (_).
    • Exemple: user_name, total_amount, car_model

    Nota: A JavaScript, aquesta convenció és menys utilitzada, però podries trobar-la en API o bases de dades.

    4. Kebab Case

    • Similar al snake_case, però utilitzant un guió (-) per separar les paraules. Aquesta convenció s’utilitza generalment en noms de fitxers, URLs o en entorns com CSS o en noms d'arxius.
    • Exemple: user-name, total-amount, car-model

    Nota: No s'usa per a noms de variables o funcions en codi JavaScript, però pot ser útil per noms de fitxers, carpetes o rutes.

    5. Constantes (constants)

    • Per a les constants que no canvien (és a dir, valors que es mantenen igual durant tota l'execució), s'usa UPPERCASE amb guions baixos per separar les paraules.
    • Exemple: MAX_SIZE, PI_VALUE, DEFAULT_LANGUAGE

    Regla: Escriu les constants en majúscules i separa les paraules amb guions baixos.

    6. Evitar abreviacions poc clares

    • És una bona pràctica evitar les abreviacions que poden ser difícils de comprendre per altres desenvolupadors. Per exemple, en lloc de str per a una cadena de text, utilitza string, i en lloc de num utilitza number.
    • Exemple: userName en comptes de usrNm, totalAmount en comptes de totAmt.

    7. Mètodes d'Acció

    • Els noms de les funcions o mètodes han de ser verbs perquè representen una acció que el codi realitza.
    • Exemple: getUserName(), calculateTotalAmount(), sendEmail()

    8. Evitar noms generals o massa genèrics

    • Intenta evitar noms com data, info, item, perquè són massa generals i no proporcionen molta informació sobre què fa la variable o funció.
    • Exemple millor: userData, orderList, productItems.

    9. Evitar noms reservats o conflictes amb paraules clau

    • Evita utilitzar paraules clau del llenguatge (com class, return, let, for, etc.) com a noms de variables o funcions.
    • Exemple: No utilitzis let, const, return com a noms de variables.

    10. Variables booleans

    • Per a variables booleans (que es poden establir com true o false), és habitual utilitzar una forma interrogativa per al nom, generalment amb un prefix com is, has, can o should.
    • Exemple: isValid, hasPermission, canEdit, shouldUpdate

    11. Evitar l'ús de _ (guió baix inicial)

    • A JavaScript, la convenció de utilitzar un guió baix inicial _ per indicar que una variable és privada és habitual en alguns codi, però no és una pràctica estàndard del llenguatge. Aquest comportament sol ser comú quan es fa servir amb class i mètodes privats.

    Exemple: _privateVar, _calculateTotal()

    12. Noms de funcions i variables en plural o singular

    • Arrays o llistes: Si una variable conté diversos elements, és habitual utilitzar noms en plural.
    • Exemple: users (per a un array de persones), items (per a una llista d'objectes).

    Resum de les convencions:

    • Camel Case: per a variables i funcions (totalAmount, calculateSum)
    • Pascal Case: per a classes i constructors (Person, CarModel)
    • Snake Case: per a noms de fitxers o en alguns entorns (per exemple, user_name)
    • Kebab Case: per a noms d'arxius i URLs (per exemple, user-name)
    • UPPERCASE: per a constants (MAX_SIZE, PI_VALUE)

    Seguir aquestes convencions ajuda a mantenir el codi net, organitzat i fàcil de llegir, especialment quan treballes en projectes col·laboratius amb altres desenvolupadors.

    Classes i herència

    A JavaScript, les classes i l'herència són mecanismes que permeten organitzar i estructurar el codi de manera més neta i reutilitzable, imitant el comportament d'altres llenguatges de programació orientats a objectes (OOP). Tot i que JavaScript no és un llenguatge estrictament orientat a objectes, la introducció de les classes en ECMAScript 6 (ES6) va fer que la programació orientada a objectes fos més accessible i intuïtiva.

    1. Classes en JavaScript

    Una classe és una plantilla o un model per crear objectes. És una forma de definir els atributs (propietats) i mètodes (funcions) d'un objecte.

    Definir una classe:

    A JavaScript, es poden definir les classes utilitzant la paraula clau class.

    class Persona {
  // Constructor: inicialitza els valors de l'objecte
  constructor(nom, edat) {
    this.nom = nom; // propietat
    this.edat = edat; // propietat
  }

  // Mètode de la classe
  saludar() {
    console.log(`Hola, em dic ${this.nom} i tinc ${this.edat} anys.`);
  }
}

// Crear una instància de la classe
const persona1 = new Persona('Joan', 30);
persona1.saludar(); // Output: Hola, em dic Joan i tinc 30 anys.

    • constructor(): El mètode especial constructor es crida automàticament quan es crea una nova instància de la classe. S'hi defineixen els atributs o propietats inicials de l'objecte.

    • Mètodes: Els mètodes es defineixen a l'interior de la classe. Aquests són simplement funcions que actuen sobre les propietats de l'objecte.

    3. Herència en JavaScript

    L'herència permet crear una nova classe (subclasse) a partir d'una classe existent (superclasse). La subclasse heretarà totes les propietats i mètodes de la superclasse i pot afegir-hi les seves pròpies.

    Sintaxi d'herència:

    A JavaScript, per crear una subclasse, s'utilitza la paraula clau extends.

    class Treballador extends Persona {
  constructor(nom, edat, feina) {
    // Cridem el constructor de la superclasse (Persona)
    super(nom, edat);  // super() crida al constructor de la classe pare
    this.feina = feina; // nova propietat per a la subclasse
  }

  // Mètode de la subclasse
  treballar() {
    console.log(`${this.nom} està treballant com a ${this.feina}.`);
  }
}

const treballador1 = new Treballador('Pere', 40, 'enginyer');
treballador1.saludar(); // Output: Hola, em dic Pere i tinc 40 anys.
treballador1.traballar(); // Output: Pere està treballant com a enginyer.

    Explicació de l'herència:

    • extends: Permet a la subclasse heretar les propietats i mètodes de la classe pare.

    • super(): Es crida al constructor de la classe pare. És necessari per inicialitzar les propietats de la classe pare abans de definir les pròpies de la subclasse.

    • Subclasse: Pot afegir mètodes propis a més dels mètodes heretats de la classe pare.

    • Classes: Són plantilles per crear objectes amb propietats i mètodes. S'utilitza class per definir-les.
    • Herència: Permet crear subclasses que hereten propietats i mètodes de les classes pare. S'utilitza extends per crear una subclasse.
    • Sobreescriptura de mètodes: Les subclasses poden redefinir els mètodes heretats.
    • Mètodes estàtics: Pertanyen a la classe en lloc de les instàncies.
    • Getters i Setters: Permeten accedir i modificar propietats de manera controlada.

    L'ús de classes i herència facilita l'estructuració de codi en JavaScript, fent-lo més modular, reutilitzable i net.

    Entrada/eixida en node.js

    El mòdul readline de Node.js és una eina poderosa per manejar entrades i sortides de la línia de comandes de manera interactiva. Aquest mòdul permet llegir línies d'entrada del teclat (com a una aplicació de consola) i gestionar l'output (sortida) a la terminal.

    Aquí et mostraré com utilitzar readline per llegir i escriure informació a la línia de comandes amb exemples simples.

    1. Configurar readline per llegir dades de l'usuari

    Per començar, primer hem d'importar el mòdul readline i configurar un interface per la lectura i l'escriptura.

    JavaScript

    const readline = require('readline');

// Crear una interfície per llegir de la consola (entrada) i escriure a la consola (sortida)
const rl = readline.createInterface({
  input: process.stdin,  // Entrada des de la línia de comandes
  output: process.stdout  // Sortida a la línia de comandes
});

    2. Llegir l'entrada de l'usuari

    Un cop tinguem l'interfície configurada, podem utilitzar el mètode question per fer preguntes a l'usuari i capturar les respostes.

  console.log(`Hola, ${nom}!`);

    • rl.close() tanca l'interfície un cop hem acabat de llegir les dades.

    const readline = require('readline');

// Crear la interfície readline
const rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout
});

// Funció per fer preguntes de manera asíncrona amb promeses
function preguntar(pregunta) {
  return new Promise((resolve, reject) => {
    rl.question(pregunta, (resposta) => {
      resolve(resposta);
    });
  });
}

// Funció principal asíncrona
async function obtenirInformacio() {
  const nom = await preguntar('Com et dius? ');
  const edat = await preguntar('Quants anys tens? ');

  console.log(`Hola, ${nom}, tens ${edat} anys.`);

  // Tanquem la interfície
  rl.close();
}

obtenirInformacio();

    Explicació:

    • La funció preguntar retorna una promesa que es resol amb la resposta de l'usuari.
    • A la funció obtenirInformacio, utilitzem await per esperar que cada pregunta sigui contestada abans de passar a la següent.
    • rl.close() es crida un cop acabem de fer totes les preguntes.

    5. Maneig d'entrada i sortida per línies

    A més de question, també podem llegir línies de manera contínua (per exemple, fins que l'usuari premi el teclat Ctrl + C o escrigui una línia buida).

// Llegeix cada línia d'entrada de l'usuari fins que el procés sigui tancat

    Explicació:

    • Quan l'usuari prem Ctrl+C, l'esdeveniment close es dispara i el procés finalitza.

    • Entrada de dades: El mòdul readline s'utilitza per llegir dades de la línia de comandes.
    • Sortida de dades: Es pot utilitzar per mostrar missatges a la terminal.
    • Gestió de preguntes: Amb question podem fer preguntes interactives.
    • Línies contínues: Usant l'esdeveniment line, podem llegir diverses línies d'entrada.
    • Manejo amb Promeses i async/await: Per facilitar la lectura del codi i la gestió de fluxos asíncrons, podem utilitzar Promeses i async/await.

    Importació exportació

    A JavaScript, l'exportació i la importació de mòduls és una pràctica essencial per organitzar el codi en diferents fitxers i permetre'n la reutilització. Això ajuda a mantenir el codi modular, llegible i fàcil de mantenir.

    JavaScript té diferents maneres d'exportar i importar mòduls. La més comuna és a través dels mòduls ES6 (import/export), que és el mètode estàndard per treballar amb mòduls a JavaScript modern. A més, també existeix el sistema de mòduls de CommonJS, que és el que s'utilitza principalment en Node.js.

    1. Mòduls ES6 (import/export)

    Els mòduls ES6 són la forma estàndard d'exportar i importar mòduls a JavaScript. Aquest sistema és compatible amb la majoria de navegadors moderns i es pot utilitzar també en Node.js amb l'extensió .mjs o habilitant el suport a mòduls a través de la configuració del package.json.

    Exportar des d'un mòdul (fitxer .js):

    Hi ha dues maneres d'exportar entitats (variables, funcions, classes, etc.) d'un mòdul: exportació anònima i exportació anomenada.

    Exportació anònima: Es fa servir per exportar només una entitat per mòdul. Pots exportar una variable, funció o classe directament quan la defines.

    JavaScript

// Exportació d'una funció
export function suma(a, b) {
  return a + b;
}

// Exportació de múltiples entitats
const pi = 3.1416;
export { pi };

    Exportació anomenada: També es poden exportar múltiples entitats d'un mateix mòdul.

    JavaScript
    1

const suma = (a, b) => a + b;

export { pi, suma };

    També pots fer una exportació anomenada més compacta en el mateix moment de la declaració:

    JavaScript
    1
2
3

export const suma = (a, b) => a + b;

    Importar un mòdul (fitxer .js):

    Per importar els mòduls en un altre fitxer, utilitzem import per a les exportacions anomenades i import també per a l'exportació per defecte.

    Importació d'exportacions anomenades:

    JavaScript

console.log(pi);  // Output: 3.1416

    Importació d'exportació per defecte:

    Si només s'exporta una entitat (per exemple, una funció o una classe), es pot utilitzar l'exportació per defecte.

    Importar tot el mòdul: Si vols importar tot el contingut del mòdul i utilitzar-lo sota un únic nom, pots fer-ho així:

    A Node.js, el sistema de mòduls més antic utilitza CommonJS. En aquest sistema, les entitats es poden exportar amb module.exports i importar amb require().

    3. Com utilitzar import/export a Node.js?

    Si utilitzes Node.js, de forma predeterminada, es fa servir CommonJS. No obstant això, a partir de Node.js 14 i amb l'ús de l'extensió .mjs o habilitant "type": "module" al fitxer package.json, pots utilitzar mòduls ES6.

    Exemple de configuració per a Node.js (mòduls ES6):

    1. Afegeix "type": "module" al teu fitxer package.json:
    JSON
    1
2
3
    {
  "type": "module"
}
    1. Després, pots utilitzar import i export a Node.js:
    JavaScript
    1
2
3
4

7
8
9
    // fitxer 'math.mjs'

  return a + b;
}

// fitxer 'app.mjs'
import { suma } from './math.mjs';

console.log(suma(2, 3));  // Output: 5

    4. Diferències entre import/export i require/module.exports

    • Sintaxi: import/export és la sintaxi estàndard per a JavaScript modern, mentre que require/module.exports és la sintaxi utilitzada per CommonJS, el sistema original de mòduls de Node.js.
    • Asíncrones: require() es carrega de manera sincrònica, mentre que import es fa de manera asíncrona (pot ser important quan es treballa amb aplicacions web).
    • Compatibilitat: import/export és compatible amb els navegadors moderns i la majoria d'entorns que implementen JavaScript, mentre que require() és específic de Node.js.

    5. Exportació i importació per defecte

    Quan utilitzes exportació per defecte, només es pot exportar una sola entitat, i pots importar-la sense utilitzar claus.

    Exemple d'exportació per defecte i importació:

    Exportació per defecte:

  return a + b;

    Importació per defecte:

    JavaScript
    1
2
3

    Resum:

    1. ES6 (import/export): Utilitzat per a mòduls moderns a JavaScript. Utilitza export per exportar i import per importar mòduls.
    2. CommonJS (require/module.exports): Utilitzat principalment a Node.js. module.exports s'utilitza per exportar, i require() per importar.
    3. Node.js i mòduls ES6: A partir de Node.js 14, pots utilitzar mòduls ES6 si afegeixes "type": "module" al fitxer package.json o utilitzes l'extensió .mjs.
    4. Exportacions per defecte: S'exporta una sola entitat per mòdul i s'importa sense claus.

    Les convencions d'exportació i importació permeten estructurar el teu codi en mòduls reutilitzables, millorant la llegibilitat i la mantenibilitat del projecte.

    Funcions sobre arrays

    Les funcions map(), filter() i reduce() són mètodes molt utilitzats en JavaScript per treballar amb arrays de manera funcional. Aquestes funcions ajuden a transformar, filtrar i reduir dades en un array de manera concisa i expressiva, sense la necessitat d’utilitzar bucles explícits. Són molt útils quan volem treballar amb dades de manera immutables (sense modificar l'array original).

    1. map()

    La funció map() s'utilitza per transformar els elements d'un array. Aquesta funció aplica una funció a cada element de l'array i retorna un nou array amb els resultats.

    Sintaxi:

    JavaScript
    1
    array.map(callback(currentValue, index, array))
    • callback: És una funció que s'aplica a cada element de l'array.
    • currentValue: L'element actual que s'està processant.
    • index (opcional): L'índex de l'element actual.
    • array (opcional): L'array original.

    Exemple:

    Suposem que volem multiplicar cada número d'un array per 2:

    JavaScript
    1
2
3
4
    const nombres = [1, 2, 3, 4, 5];
const multiplicats = nombres.map(num => num * 2);

console.log(multiplicats);  // Output: [2, 4, 6, 8, 10]

    En aquest exemple, map() aplica la funció num => num * 2 a cada element de l'array nombres, retornant un nou array amb els valors multiplicats per 2.

    2. filter()

    La funció filter() s'utilitza per filtrar elements d'un array segons una condició. Aquesta funció retorna un nou array amb només els elements que compleixen la condició especificada a la funció de callback.

    Sintaxi:

    JavaScript
    1
    array.filter(callback(currentValue, index, array))
    • callback: Funció que defineix la condició per filtrar els elements.
    • currentValue: L'element actual que s'està processant.
    • index (opcional): L'índex de l'element actual.
    • array (opcional): L'array original.

    Exemple:

    Suposem que volem filtrar els nombres imparells d'un array:

    JavaScript
    1
2
3
4
    const nombres = [1, 2, 3, 4, 5];
const imparells = nombres.filter(num => num % 2 !== 0);

console.log(imparells);  // Output: [1, 3, 5]

    En aquest exemple, filter() filtra els elements que compleixen la condició num % 2 !== 0 (és a dir, els imparells).

    3. reduce()

    La funció reduce() s'utilitza per reducir un array a un sol valor. La funció de callback que es passa a reduce() agafa dos valors a cada iteració: un acumulador (el valor que es va construint) i el valor actual de l'array.

    Sintaxi:

    JavaScript
    1
    array.reduce(callback(accumulator, currentValue, index, array), initialValue)
    • callback: Funció que es crida per a cada element.
    • accumulator: L'acumulador (el valor que es va construint a mesura que recorrem l'array).
    • currentValue: El valor actual de l'element de l'array.
    • initialValue (opcional): Valor inicial per a l'acumulador (si no es passa, el primer element de l'array s'utilitza com a valor inicial).

    Exemple:

    Suposem que volem sumar tots els elements d'un array:

    JavaScript
    1
2
3
4
    const nombres = [1, 2, 3, 4, 5];
const suma = nombres.reduce((acc, num) => acc + num, 0);

console.log(suma);  // Output: 15

    En aquest exemple, reduce() acumula els valors de l'array nombres sumant-los a l'acumulador acc. El valor inicial de l'acumulador és 0.

    Exemple amb objectes:

    Imagina que tens un array de persones i vols obtenir l'edat total de totes les persones:

    JavaScript
    1
2
3
4
5
6
7
8
9
    const persones = [
  { nom: 'Joan', edat: 30 },
  { nom: 'Maria', edat: 25 },
  { nom: 'Pere', edat: 35 }
];

const edatTotal = persones.reduce((acc, persona) => acc + persona.edat, 0);

console.log(edatTotal);  // Output: 90

    Aquí, reduce() acumula les edats de les persones en l'acumulador acc, retornant la suma total de les edats.

    Resum de les funcions:

    Funció Descripció Exemple
    map() Aplica una funció a cada element d'un array i retorna un nou array. [1, 2, 3].map(num => num * 2) → [2, 4, 6]
    filter() Filtra els elements d'un array segons una condició i retorna un array amb els elements que compleixen la condició. [1, 2, 3, 4].filter(num => num % 2 === 0) → [2, 4]
    reduce() Redueix un array a un sol valor, acumulant els resultats a través d'un procés iteratiu. [1, 2, 3].reduce((acc, num) => acc + num, 0) → 6

    Casos d'ús comunes:

    • map(): Quan necessitem transformar cada element d'un array (per exemple, aplicar una operació matemàtica o modificar l'estructura de cada element).
    • filter(): Quan necessitem filtrar elements d'un array basant-nos en alguna condició (per exemple, obtenir un subconjunt d'elements que compleixen una regla específica).
    • reduce(): Quan necessitem reduir un array a un únic valor (per exemple, la suma total d'una llista de números, la concatenació de textos, o la creació d'un objecte).

    Aquestes funcions permeten escriure codi més net i funcional, evitant l'ús de bucles for explícits i fent el codi més llegible.
