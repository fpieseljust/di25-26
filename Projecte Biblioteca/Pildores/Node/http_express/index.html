    Mètodes HTTP i Express

    És important entendre les bases dels mètodes HTTP per començar a desenvolupar serveis, ja que són la forma en què el client i el servidor es comuniquen per realitzar operacions sobre recursos (com llibres en una biblioteca).

    Mètodes HTTP principals:

    1. GET: S'utilitza per obtenir dades del servidor. Aquest mètode no modifica cap dada. És el més habitual quan es vol obtenir informació d'un recurs.

    2. Exemple en Express:

      JavaScript
      1
2
3
4
      app.get('/llibres', (req, res) => {
    // Codi per obtenir tots els llibres
    res.send('Llista de llibres');
});

    3. POST: S'utilitza per enviar dades al servidor per crear un recurs. El servidor processa les dades i, normalment, les afegeix a la base de dades.

    4. Exemple en Express:

      JavaScript
      1
2
3
4
      app.post('/llibres', (req, res) => {
    // Codi per crear un llibre
    res.send('Llibre creat');
});

    5. PUT: S'utilitza per actualitzar un recurs existent. Reemplaça el recurs completament amb les noves dades.

    6. Exemple en Express:

      JavaScript
      1
2
3
4
      app.put('/llibres/:id', (req, res) => {
    // Codi per actualitzar un llibre específic
    res.send('Llibre actualitzat');
});

    7. PATCH: Similar al PUT, però s'utilitza per actualitzar només una part d'un recurs (no substitueix completament les dades).

    8. Exemple en Express:

      JavaScript
      1
2
3
4
      app.patch('/llibres/:id', (req, res) => {
    // Codi per actualitzar parcialment un llibre
    res.send('Llibre parcialment actualitzat');
});

    9. DELETE: S'utilitza per eliminar un recurs específic.

    10. Exemple en Express:

      JavaScript
      1
2
3
4
      app.delete('/llibres/:id', (req, res) => {
    // Codi per eliminar un llibre
    res.send('Llibre eliminat');
});

    11. HEAD: Similar a GET, però només retorna els headers, no el cos de la resposta. S'utilitza per obtenir metadades sense transferir el cos de la resposta.

    12. Express no té un mètode específic per HEAD, però es pot gestionar amb app.head() de la mateixa manera que app.get().

    13. OPTIONS: S'utilitza per obtenir informació sobre les opcions de comunicació disponibles per a un recurs, com els mètodes HTTP suportats. És útil per conèixer què pot fer un client amb el servidor.

    14. Exemple en Express:

      JavaScript
      1
2
3
4
      app.options('/llibres', (req, res) => {
    res.set('Allow', 'GET, POST, OPTIONS');
    res.send();
});

    Relació amb Express:

    • GET, POST, PUT, PATCH, DELETE: Són directament gestionats a Express mitjançant els mètodes app.get(), app.post(), app.put(), app.patch(), i app.delete().
    • HEAD i OPTIONS: Són menys comuns, però també poden ser utilitzats mitjançant app.head() i app.options() si és necessari.

    const express = require('express');
const app = express();
app.use(express.json()); // Per processar les dades JSON

// GET: Obtenir llistat de llibres
app.get('/llibres', (req, res) => {
    res.send('Llista de llibres');
});

// POST: Crear un llibre
app.post('/llibres', (req, res) => {
    const llibre = req.body;
    // Afegir llibre a la base de dades
    res.status(201).send('Llibre creat');
});

// PUT: Actualitzar un llibre
app.put('/llibres/:id', (req, res) => {
    const llibreId = req.params.id;
    const llibreNou = req.body;
    // Actualitzar el llibre a la base de dades
    res.send(`Llibre amb id ${llibreId} actualitzat`);
});

// DELETE: Eliminar un llibre
app.delete('/llibres/:id', (req, res) => {
    const llibreId = req.params.id;
    // Eliminar el llibre de la base de dades
    res.send(`Llibre amb id ${llibreId} eliminat`);
});

app.listen(3000, () => {
    console.log('API de la biblioteca en funcionament a http://localhost:3000');
});

    Aquest seria un exemple bàsic de com es poden gestionar les operacions CRUD d'una biblioteca utilitzant mètodes HTTP a Express.
